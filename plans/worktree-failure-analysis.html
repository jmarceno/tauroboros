Fix Strategy

Recommended solutions to address each root cause and prevent future occurrences.

1

Atomic Worktree Creation

Make worktree creation transactional. Use temp directory first, only move to final location after successful git worktree add.

2

Pre-Container Verification

Verify worktree is fully populated before mounting in container. Check for .git file and expected file structure.

3

Timeout with Completion Detection

If task has substantial completion (e.g., >30K events), trigger merge on timeout rather than hard failure.

4

Orphan Cleanup on Failure

If git worktree add fails, immediately remove the created directory to prevent orphan accumulation.

Implementation Priority: Start with #4 (orphan cleanup) as it's the fastest fix to prevent directory accumulation. Then implement #1 (atomic creation) for robustness. #3 (timeout handling) is important for long-running tasks. #2 (verification) adds safety but may be redundant after #1 is in place.

Key Insight

The root cause is a non-atomic worktree creation process that leaves orphaned directories when git commands fail. These orphans block subsequent attempts ("WORKTREE_ALREADY_EXISTS"), eventually causing timeout failures that cascade through the dependency graph. The fix requires making worktree creation transactional: succeed completely or clean up completely, with no intermediate states.
