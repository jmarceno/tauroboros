Planning Chat Enhancements

Four feature implementations for improved user experience: clipboard image paste, agent working indicators, mermaid diagram maximization, and session renaming.

Clipboard Image Paste
Allow users to paste images from clipboard directly into planning chat. Convert to base64 and send as attachments.
Agent Working Indicator
Show clear indicator when Pi AI is actively working. Track agent_start, tool_execution_start, and agent_end events.
Maximize Mermaid Diagrams
Add maximize/expand button to mermaid diagrams. View in fullscreen modal for better visibility.
Rename Planning Sessions
Allow users to rename planning chat sessions. Add name field to workflow_sessions table and UI for editing.

API Effect Pattern 
// Frontend API wrapper using Effect
export const updateSessionName = (
  sessionId: string,
  name: string
): Effect.Effect<void, ApiError> => {
  return Effect.tryPromise({
    try: async () => {
      const response = await fetch(
        `/api/planning/sessions/${sessionId}/name`,
        {
          method: "PUT",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ name }),
        }
      )
      if (!response.ok) {
        throw new Error("Failed to rename session")
      }
    },
    catch: (error) =>
      new ApiError({
        message: error instanceof Error ? error.message : "Unknown error",
      }),
  })
}

router.put("/api/planning/sessions/:id/name", ({ params, req, json, broadcast, db }) =>
  Effect.gen(function* () {
    const session = db.getWorkflowSession(params.id)
    if (!session) {
      return yield* Effect.fail(notFoundError("Session not found"))
    }

    const body = (yield* Effect.tryPromise({
      try: () => req.json() as Promise<{ name?: string }>,
      catch: (cause) => badRequestError("Invalid JSON"),
    }))

    if (!body.name || body.name.trim() === "") {
      return yield* Effect.fail(badRequestError("Name is required"))
    }

    const updated = db.updateWorkflowSession(params.id, {
      name: body.name.trim(),
    })

    broadcast({ type: "planning_session_updated", payload: updated })
    return json(updated)
  }),
)

Clipboard Image Paste - Technical Details

Listen for paste event on textarea, check for e.clipboardData.files. Images are processed through a pipeline: resized to 1024px on the largest dimension (maintaining aspect ratio), encoded as JPEG at 90% quality, then converted to base64 for transmission. Show thumbnail preview in attachment list.

Image Processing Pipeline
async function processImage(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const img = new Image()
    const canvas = document.createElement('canvas')
    const ctx = canvas.getContext('2d')
    
    img.onload = () => {
      const MAX_SIZE = 1024
      let { width, height } = img
      
      // Resize maintaining aspect ratio
      if (width > height && width > MAX_SIZE) {
        height = (height * MAX_SIZE) / width
        width = MAX_SIZE
      } else if (height > MAX_SIZE) {
        width = (width * MAX_SIZE) / height
        height = MAX_SIZE
      }
      
      canvas.width = width
      canvas.height = height
      ctx?.drawImage(img, 0, 0, width, height)
      
      // Encode as JPEG 90% quality, convert to base64
      const base64 = canvas.toDataURL('image/jpeg', 0.9)
      resolve(base64)
    }
    
    img.onerror = reject
    img.src = URL.createObjectURL(file)
  })
}

Risk Assessment & Mitigation

Potential Risks
  • Base64 images may exceed message size limits
  • Agent events may fire rapidly - need debouncing
  • Multiple sessions renaming simultaneously could cause race conditions
  • Mermaid modal needs to handle large diagrams
Mitigation Strategies
  • Limit images to 1MB, compress if larger
  • Use SolidJS createEffect with proper cleanup
  • Use unique request IDs for idempotency
  • Virtualize large diagrams, add zoom controls
